store the min (3) in a root, then continue with 5, 8, 10, 20, 50, 80

Goal: Optimize the lookup operation, not the insert time.



This will keep the tree balanced on L and R, making the lookup operation optimized.

Need to calculate the equilibrium (balance) factor - BF for each node



Heigh(Left/Right Subtree):

GE = H(LS) - H(RS) (or viceversa)

GE belongs to {-1, 0, 1}



Height computation:

if H(AVL\* root) return (root == NULL) ? 0 : 1 + max(H(root->left), H(root->right));

O(log(n)) will be kept for the lookup function as well as the insertion and deletion operations

since O(log(n)) + c = O(log(n))
